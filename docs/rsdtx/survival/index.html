Survival Analysis with Kaplan-Meier

The Problem

  • In a clinical trial some patients experience the event of interest (death, relapse, recovery) while others leave the study before the trial ends or are still event-free when the data are analysed.
  • An observation where the true event time is unknown because we only know the patient survived past a certain point is called right-censored.
  • Ignoring censored observations or treating them as events both bias the estimate of survival time.
  • The Kaplan-Meier estimator (1958) accounts for censoring by updating the survival probability only at observed event times, using only the patients still under observation.

The Survival Function

  • The survival function $S(t)$ is the probability that a randomly chosen individual survives past time $t$:

$$S(t) = P(T > t)$$

  • $S(0) = 1$ (everyone is alive at the start) and $S(t)$ is non-increasing.
  • For an Exponential distribution with mean $\mu$, $S(t) = e^{-t/\mu}$ and the median survival time is $\mu \ln 2$.

The survival time of a population follows an Exponential distribution with mean $\mu = 20$ days. Using $S(t) = e^{-t/\mu}$, compute $S(20 \ln 2)$ to three decimal places.

The Kaplan-Meier Estimator

  • At each unique event time $t_i$, let $n_i$ be the number still at risk (not yet had event, not yet censored) and $d_i$ the number who experience the event at exactly $t_i$.
  • The product-limit estimate multiplies successive conditional survival factors:

$$\hat{S}(t) = \prod_{t_i \leq t} \left(1 - \frac{d_i}{n_i}\right)$$

  • Censored observations reduce the risk set $n_i$ for all subsequent event times but do not count as events — this is the key difference from simply computing the empirical CDF.
  • Tied censoring and event times: by convention, censoring at time $t$ is treated as occurring just after any events at $t$, so the censored patient is still in $n_i$ for that event time.

Patient A is censored at day 10 (last known to be alive). Patient B has an event at day 12. How does patient A appear in the Kaplan-Meier calculation at $t = 12$?

Patient A contributes $d_i = 1$ at $t = 12$ because they represent an unknown event.
Wrong: censoring means the event did not occur before day 10; the patient is not counted as an event.
Patient A is not in $n_i$ at $t = 12$ because they left the study before that time.
Correct: once censored at day 10, patient A is no longer at risk and is excluded from $n_i$ at all subsequent times.
Patient A is counted in $n_i$ at $t = 12$ but contributes $d_i = 0$.
Wrong: patient A is not at risk at day 12 — they cannot contribute to the risk set after being censored at day 10.
Patient A's censoring time is used as their event time.
Wrong: substituting the censoring time as an event time is the bias the KM estimator is specifically designed to avoid.

Generating Synthetic Data

  • Each patient's true event time is drawn from Exponential($\mu = 20$ days).
  • An independent censoring time drawn from Uniform($10$, $35$) days simulates patients who leave the study at different points.
  • The observed time is whichever comes first; observed = 1 if the event came first.
SEED = 7493418  # RNG seed for reproducibility
N_PATIENTS = 100  # number of patients in the study
MEAN_SURVIVAL = (
    20.0  # mean event time drawn from Exponential(scale=MEAN_SURVIVAL) in days
)
# Censoring time drawn from Uniform(CENSOR_MIN, CENSOR_MAX): patients followed up for a random
# duration, with the study ending if the event has not yet occurred.
CENSOR_MIN = 10.0  # minimum follow-up time (days)
CENSOR_MAX = 35.0  # maximum follow-up time (days)

def make_survival_data(
    n=N_PATIENTS,
    mean_survival=MEAN_SURVIVAL,
    censor_min=CENSOR_MIN,
    censor_max=CENSOR_MAX,
    seed=SEED,
):
    """Return a Polars DataFrame with columns 'time' and 'observed'.

    Each patient has a true event time drawn from Exponential(scale=mean_survival).
    An independent censoring time is drawn from Uniform(censor_min, censor_max).
    The observed time is whichever comes first.
    'observed' is 1 if the event occurred before censoring, 0 otherwise.
    """
    rng = np.random.default_rng(seed)
    true_event_times = rng.exponential(scale=mean_survival, size=n)
    censoring_times = rng.uniform(low=censor_min, high=censor_max, size=n)
    observed = (true_event_times <= censoring_times).astype(int)
    times = np.minimum(true_event_times, censoring_times)
    return pl.DataFrame({"time": times, "observed": observed})

Put these Kaplan-Meier steps in the correct order.

  1. Sort all unique times at which events (not censorings) occurred
  2. For each event time, count $n_i$ (at risk) and $d_i$ (events)
  3. Compute the conditional survival factor $(1 - d_i / n_i)$ at each event time
  4. Multiply the factors cumulatively to obtain $\hat{S}(t)$
  5. Report the first $t$ where $\hat{S}(t) \leq 0.5$ as the estimated median

Computing the Estimate

def kaplan_meier(times, observed):
    """Compute the Kaplan-Meier product-limit survival estimate.

    At each unique event time t_i:
      - n_i = number of patients still at risk (time >= t_i, not yet censored)
      - d_i = number who experienced the event at exactly t_i
      - S(t_i) = S(t_{i-1}) * (1 - d_i / n_i)

    Returns (event_times, survival_probs) where both arrays are sorted in
    ascending order and have the same length.
    """
    times = np.asarray(times, dtype=float)
    observed = np.asarray(observed, dtype=int)
    event_times = np.sort(np.unique(times[observed == 1]))
    surv = 1.0
    survival_probs = []
    for t in event_times:
        n_at_risk = np.sum(times >= t)
        n_events = np.sum((times == t) & (observed == 1))
        surv *= 1.0 - n_events / n_at_risk
        survival_probs.append(surv)
    return event_times, np.array(survival_probs)
  • The inner loop over event_times (sorted unique times where observed == 1) ensures that only true event times, not censoring times, trigger a drop in $\hat{S}$.
  • np.sum(times >= t) counts all patients still at risk at time $t$: those whose recorded time (event or censoring) is at least $t$.

Median Survival Time

def median_survival(event_times, survival_probs):
    """Return the smallest event time at which the survival estimate drops to 0.5 or below.

    Returns None if the survival curve never reaches 0.5 (too many censored observations
    prevent the estimate from falling that far).
    """
    for t, s in zip(event_times, survival_probs):
        if s <= 0.5:
            return float(t)
    return None
  • The median is the first time $\hat{S}(t) \leq 0.5$; it does not require $\hat{S}$ to reach zero, making it well-defined even when many patients are censored.
  • Returning None when the curve never reaches $0.5$ is important: a function that silently returns the last event time would give a misleading result.

For an Exponential distribution with mean $\mu = 20$ days, compute the theoretical median survival time $\mu \ln 2$ in days, rounded to one decimal place.

Visualizing the Survival Curve

def plot_km(times, observed, event_times, survival_probs, filename):
    """Save a Kaplan-Meier step function with censoring tick marks.

    The step function starts at S = 1 and drops at each event time.
    Censored observations are shown as small vertical ticks at the
    survival probability current when each patient left the study.
    """
    # Build step-function coordinates: prepend t=0, S=1 so the curve
    # starts at the left edge of the plot.
    step_t = np.concatenate([[0.0], event_times])
    step_s = np.concatenate([[1.0], survival_probs])
    step_df = pl.DataFrame({"t": step_t, "S": step_s})

    step_line = (
        alt.Chart(step_df)
        .mark_line(color="steelblue", strokeWidth=2, interpolate="step-after")
        .encode(
            x=alt.X("t:Q", title="Time (days)"),
            y=alt.Y(
                "S:Q", title="Survival probability", scale=alt.Scale(domain=[0, 1])
            ),
        )
    )

    # Compute survival probability at each censored time by finding the
    # largest event_time that is less than or equal to the censored time.
    times_arr = np.asarray(times, dtype=float)
    observed_arr = np.asarray(observed, dtype=int)
    censored_times = times_arr[observed_arr == 0]
    censor_s = np.ones(len(censored_times))
    for k, ct in enumerate(censored_times):
        idx = np.searchsorted(event_times, ct, side="right") - 1
        if idx >= 0:
            censor_s[k] = survival_probs[idx]

    censor_df = pl.DataFrame({"t": censored_times, "S": censor_s})
    censor_marks = (
        alt.Chart(censor_df)
        .mark_tick(color="firebrick", size=8, thickness=1.5)
        .encode(x="t:Q", y="S:Q", tooltip=["t:Q", "S:Q"])
    )

    chart = alt.layer(step_line, censor_marks).properties(
        width=500, height=250, title="Kaplan-Meier survival estimate"
    )
    chart.save(filename)

Testing

Exact values for two patients

  • With two patients who both have events at $t = 1$ and $t = 2$ and no censoring: $\hat{S}(1) = 1/2 = 0.5$ and $\hat{S}(2) = 0.5 \times 0 = 0$.
  • These are exact rational values computed without rounding, so no tolerance is needed.

Censored observation creates no event time

  • A patient censored at $t = 1$ must produce no entry in event_times; only observed events generate drop points.

Censoring reduces the risk set

  • Three patients: censored at 1.0, event at 1.5, event at 2.0. At $t = 1.5$ only two patients are at risk (the censored one has already left), so $\hat{S}(1.5) = 1 - 1/2 = 0.5$ rather than $1 - 1/3 \approx 0.667$.

Non-increasing and bounded

  • Each factor $(1 - d_i/n_i) \in [0, 1]$, so the product can only stay the same or decrease. A test on the full synthetic dataset catches any implementation bug that produces an increase.

Median survival — known case

  • Four patients with events at 1, 2, 3, 4: $\hat{S}(2) = (3/4)(2/3) = 1/2$. The median must be exactly 2.0.
  • The tolerance of $10^{-6}$ (relative) verifies exact arithmetic, not floating-point approximation.

Median returns None when curve stays above 0.5

  • One event in ten patients gives $\hat{S}(t) = 0.9$ permanently; median_survival must return None, not 0.9 or the last event time.
import numpy as np
import pytest
from generate_survival import make_survival_data
from survival import kaplan_meier, median_survival


def test_two_events_exact_values():
    # 2 patients with events at t=1 and t=2 (no censoring).
    # At t=1: n_at_risk=2, d=1 → S(1) = 1 * (1 - 1/2) = 0.5
    # At t=2: n_at_risk=1, d=1 → S(2) = 0.5 * (1 - 1/1) = 0.0
    times = np.array([1.0, 2.0])
    observed = np.array([1, 1])
    event_times, surv = kaplan_meier(times, observed)
    assert event_times[0] == pytest.approx(1.0)
    assert event_times[1] == pytest.approx(2.0)
    assert surv[0] == pytest.approx(0.5)
    assert surv[1] == pytest.approx(0.0)


def test_censored_observation_creates_no_event():
    # First patient censored at t=1 → only one event time (t=2).
    times = np.array([1.0, 2.0])
    observed = np.array([0, 1])
    event_times, surv = kaplan_meier(times, observed)
    assert len(event_times) == 1
    assert event_times[0] == pytest.approx(2.0)


def test_censoring_reduces_risk_set():
    # 3 patients: times=[1.0, 1.5, 2.0], observed=[0, 1, 1].
    # Patient 1 is censored at t=1.0 and is no longer at risk at t=1.5.
    # At t=1.5: n_at_risk=2, d=1 → S(1.5) = 1 * (1 - 1/2) = 0.5
    # At t=2.0: n_at_risk=1, d=1 → S(2.0) = 0.5 * (1 - 1/1) = 0.0
    times = np.array([1.0, 1.5, 2.0])
    observed = np.array([0, 1, 1])
    event_times, surv = kaplan_meier(times, observed)
    assert surv[0] == pytest.approx(0.5)
    assert surv[1] == pytest.approx(0.0)


def test_survival_non_increasing():
    # S(t) must never increase: each factor (1 - d_i/n_i) is in [0, 1].
    df = make_survival_data()
    _, surv = kaplan_meier(df["time"].to_numpy(), df["observed"].to_numpy())
    assert np.all(np.diff(surv) <= 0)


def test_survival_bounded():
    # Every survival probability must lie in [0, 1].
    df = make_survival_data()
    _, surv = kaplan_meier(df["time"].to_numpy(), df["observed"].to_numpy())
    assert np.all(surv >= 0.0)
    assert np.all(surv <= 1.0)


def test_median_survival_four_patients():
    # 4 patients: events at t=1, 2, 3, 4.
    # S(1)=0.75, S(2)=0.50, S(3)=0.25, S(4)=0.00.
    # Median is the first time S <= 0.5, which is t=2.
    # S(2) calculation: 1*(1-1/4)*(1-1/3) = 0.75*0.6667 = 0.50.
    times = np.array([1.0, 2.0, 3.0, 4.0])
    observed = np.array([1, 1, 1, 1])
    event_times, surv = kaplan_meier(times, observed)
    assert surv[1] == pytest.approx(0.5, rel=1e-6)
    assert median_survival(event_times, surv) == pytest.approx(2.0)


def test_median_none_when_curve_stays_above_half():
    # With only one event out of many patients the curve stays above 0.5.
    # 10 patients: 1 event at t=1, 9 censored at t=100.
    # S(1) = 1 - 1/10 = 0.9 and the curve never drops to 0.5.
    times = np.array([1.0] + [100.0] * 9)
    observed = np.array([1] + [0] * 9)
    event_times, surv = kaplan_meier(times, observed)
    assert median_survival(event_times, surv) is None

Kaplan-Meier key terms

Right-censoring
An observation where the event has not occurred by the end of follow-up; the true event time is unknown but greater than the censoring time
Survival function $S(t)$
The probability that a randomly chosen individual has not yet experienced the event by time $t$; always starts at 1 and is non-increasing
At-risk set $n_i$
The number of individuals who have neither experienced the event nor been censored before time $t_i$; the denominator in the KM update
Product-limit formula
$\hat{S}(t) = \prod_{t_i \leq t}(1 - d_i/n_i)$; censoring shrinks $n_i$ without adding to $d_i$, propagating uncertainty forward without biasing the estimate
Median survival time
The first time $\hat{S}(t) \leq 0.5$; preferred over the mean because the mean requires $S$ to reach zero, which may not occur when censoring is heavy

Exercises

Greenwood's formula

The standard error of $\hat{S}(t)$ is given by Greenwood's formula:

$$\widehat{\text{Var}}[\hat{S}(t)] = \hat{S}(t)^2 \sum_{t_i \leq t} \frac{d_i}{n_i(n_i - d_i)}$$

Add a function greenwood_se(event_times, survival_probs, n_at_risk, d_events) that returns the pointwise standard error at each event time, and plot 95% confidence bands around the survival curve.

Log-rank test

Given two groups (e.g. treatment vs. control), the log-rank test checks whether their survival curves differ significantly. At each event time $t_i$ (pooled across groups), compute the expected number of events in group 1 as $e_{1i} = n_{1i} d_i / n_i$. The test statistic is:

$$\chi^2 = \frac{(\sum_i d_{1i} - \sum_i e_{1i})^2}{\sum_i e_{1i}(1 - n_{1i}/n_i)(n_i - d_i)/(n_i - 1)}$$

Generate two groups with different mean survival times and verify that the test correctly rejects the null hypothesis at the 5% level.

Exponential parametric fit

The non-parametric KM estimate can be compared with a parametric Exponential fit. The maximum-likelihood estimate of the rate $\lambda$ is:

$$\hat{\lambda} = \frac{\text{number of events}}{\text{total person-time}} = \frac{\sum_i d_i}{\sum_i t_i}$$

Plot the KM curve and the fitted Exponential curve on the same axes. How well does the Exponential model fit the synthetic data generated from it?

Competing risks

Modify make_survival_data to generate two competing event types (e.g. relapse vs. treatment toxicity), each with its own Exponential rate. Plot cause-specific survival curves for each event type, treating the other type as a censoring event. Show that the sum of all cause-specific incidences equals the overall event rate.
