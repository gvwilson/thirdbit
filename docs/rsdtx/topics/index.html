Topic Modeling a Document Corpus

The Problem

  • A corpus of documents often covers several recurring themes, but labeling each document by hand is slow and subjective.
  • Topic modeling discovers those themes automatically by finding groups of words that tend to appear together across documents.
  • Latent Dirichlet Allocation (LDA) is a probabilistic model that represents each document as a mixture of topics and each topic as a distribution over words.
  • The approach here:
  • Generate a synthetic bag-of-words corpus in which each document is produced by a dominant topic.
  • Run collapsed Gibbs sampling to infer topic assignments for each word token.
  • Report the top words per topic and visualize document-topic proportions.

Why is LDA called an "unsupervised" method?

Because it requires labeled training documents to identify topics.
Wrong: LDA discovers topics from the text alone without any labels; that is precisely why it is called unsupervised.
Because it infers topic structure from the documents themselves, with no
pre-specified topic labels.
Correct: the model treats topic assignments for each word token as hidden variables and infers them from patterns of word co-occurrence.
Because it only works on documents where the topics are already known.
Wrong: the method is designed for cases where topics are unknown; if topics were known, supervised classification would be used instead.
Because it ignores word order within a document.
Wrong: ignoring word order (the bag-of-words assumption) is a design choice, not the reason for the "unsupervised" label.

Latent Dirichlet Allocation

  • LDA assumes the following generative process for a corpus of $D$ documents, $K$ topics, and a vocabulary of $V$ words:
  • For each topic $k$: draw word distribution $\phi_k \sim \text{Dirichlet}(\beta)$.
  • For each document $d$: draw topic distribution $\theta_d \sim \text{Dirichlet}(\alpha)$.
  • For each word position in $d$: draw topic $z \sim \theta_d$, then word $w \sim \phi_z$.
  • $\alpha$ controls how concentrated document-topic distributions are: small $\alpha$ encourages each document to focus on few topics.
  • $\beta$ controls topic-word distributions: small $\beta$ encourages each topic to use few words.
  • In practice, $\phi$ and $\theta$ are never computed directly; they are integrated out in collapsed Gibbs sampling.

Collapsed Gibbs Sampling

  • Collapsed Gibbs sampling iterates over every word token and resamples its topic assignment conditioned on all other assignments.
  • Let $n_{dk}$ be the number of tokens in document $d$ assigned to topic $k$, $n_{kw}$ the number of tokens of word $w$ assigned to topic $k$, and $n_k$ the total tokens assigned to topic $k$, all excluding the current token.
  • The conditional probability for token $i$ (document $d$, word $w$) is:

$$P(z_i = k \mid \mathbf{z}{-i}, \mathbf{w}) \propto \frac{(n$$} + \alpha)(n_{kw} + \beta)}{n_k + V\beta

  • The numerator combines how much document $d$ already favors topic $k$ (the $n_{dk}$ term) with how strongly topic $k$ is associated with word $w$ (the $n_{kw}$ term).
  • The denominator normalizes for the total size of topic $k$.
  • After many sweeps, the counts stabilize around the posterior distribution over topic assignments.

Generating Synthetic Data

  • Each of the three topics owns ten consecutive words from the 30-word vocabulary.
  • Words from a topic's own set are sampled with weight 20; all other words receive weight 0.5, so about 95% of each document's tokens come from one topic.
  • true_topic records which topic generated each document and is used only in tests.
def make_corpus(
    n_topics=N_TOPICS,
    docs_per_topic=DOCS_PER_TOPIC,
    tokens_per_doc=TOKENS_PER_DOC,
    vocab_size=VOCAB_SIZE,
    topic_weight=TOPIC_WEIGHT,
    background_weight=BACKGROUND_WEIGHT,
    seed=SEED,
):
    """Return a Polars DataFrame with columns doc_id, true_topic, word.

    Each of the n_topics topics owns vocab_size // n_topics consecutive words
    with probability topic_weight; all other words receive background_weight.
    true_topic records which topic generated each document and is used only in
    tests, not by the unsupervised sampler.
    """
    rng = np.random.default_rng(seed)
    words_per_topic = vocab_size // n_topics
    word_forms = [f"w{i:02d}" for i in range(vocab_size)]

    records = []
    doc_id = 0
    for k in range(n_topics):
        probs = np.full(vocab_size, background_weight)
        start = k * words_per_topic
        end = start + words_per_topic
        probs[start:end] = topic_weight
        probs /= probs.sum()

        for _ in range(docs_per_topic):
            sampled = rng.choice(vocab_size, size=tokens_per_doc, p=probs)
            for word_idx in sampled:
                records.append(
                    {
                        "doc_id": doc_id,
                        "true_topic": k,
                        "word": word_forms[int(word_idx)],
                    }
                )
            doc_id += 1
    return pl.DataFrame(records)

Initializing Topic Assignments

  • The sampler starts by randomly assigning each token to a topic uniformly at random.
  • The count matrices are built in a single pass over all tokens.
  • Correct initialization guarantees that $n_k = \sum_w n_{kw} = \sum_d n_{dk}$ at every point during sampling.
def init_lda(doc_ids, word_ids, n_docs, n_topics, vocab_size, rng):
    """Randomly assign topics to all tokens and build count matrices.

    Returns (topic_ids, n_dk, n_kw, n_k):
      topic_ids  1-D integer array, one entry per token, random in [0, n_topics)
      n_dk       (n_docs, n_topics) counts of topic k in document d
      n_kw       (n_topics, vocab_size) counts of word w assigned to topic k
      n_k        (n_topics,) total tokens assigned to each topic
    """
    n_tokens = len(doc_ids)
    topic_ids = rng.integers(0, n_topics, size=n_tokens)
    n_dk = np.zeros((n_docs, n_topics), dtype=int)
    n_kw = np.zeros((n_topics, vocab_size), dtype=int)
    n_k = np.zeros(n_topics, dtype=int)
    for i in range(n_tokens):
        d, w, z = int(doc_ids[i]), int(word_ids[i]), int(topic_ids[i])
        n_dk[d, z] += 1
        n_kw[z, w] += 1
        n_k[z] += 1
    return topic_ids, n_dk, n_kw, n_k

One Gibbs Step

  • A full Gibbs step visits every token in the corpus once.
  • The token is temporarily removed from the count matrices before its new topic is drawn, so its own assignment does not influence the sampling probabilities.
def gibbs_step(
    doc_ids, word_ids, topic_ids, n_dk, n_kw, n_k, alpha, beta, vocab_size, rng
):
    """Resample the topic assignment for every token (one full Gibbs sweep).

    For token i in document d with word w and current topic z:
      1. Remove the token's contribution from the count matrices.
      2. For each topic k compute the unnormalised weight:
            (n_dk[d, k] + alpha) * (n_kw[k, w] + beta) / (n_k[k] + vocab_size * beta)
      3. Normalise and draw a new topic.
      4. Restore the token under its new topic.
    Modifies topic_ids, n_dk, n_kw, and n_k in place.
    """
    n_topics = n_k.shape[0]
    for i in range(len(doc_ids)):
        d, w, z = int(doc_ids[i]), int(word_ids[i]), int(topic_ids[i])
        n_dk[d, z] -= 1
        n_kw[z, w] -= 1
        n_k[z] -= 1

        probs = (n_dk[d] + alpha) * (n_kw[:, w] + beta) / (n_k + vocab_size * beta)
        probs /= probs.sum()
        new_z = int(rng.choice(n_topics, p=probs))

        topic_ids[i] = new_z
        n_dk[d, new_z] += 1
        n_kw[new_z, w] += 1
        n_k[new_z] += 1

Why must a token's contribution be removed from the count matrices before resampling its topic?

To save memory by not storing the current assignment.
Wrong: removing the token does not save memory; the arrays remain the same size.
To avoid circular conditioning: the token's current assignment would otherwise
inflate the probability of keeping the same topic.
Correct: the conditional distribution must be computed excluding token $i$; leaving it in would bias the sampler toward its current assignment.
To allow the sampler to explore more topics per iteration.
Wrong: the number of topics explored per token is always $K$, regardless of whether the current token is removed first.
Because numpy arrays do not support in-place modification.
Wrong: numpy arrays support in-place modification; the removal and restoration are a deliberate algorithmic requirement.

Running the Sampler

def run_lda(df, n_topics, n_iter=N_ITER, alpha=ALPHA, beta=BETA, seed=SEED):
    """Run the collapsed Gibbs sampler and return (n_dk, n_kw, vocab).

    df must have columns doc_id and word.
    Returns:
      n_dk   (n_docs, n_topics) topic-count matrix (un-normalised)
      n_kw   (n_topics, vocab_size) word-count matrix (un-normalised)
      vocab  sorted list of word strings, length vocab_size
    """
    vocab = sorted(df["word"].unique().to_list())
    word_to_id = {w: i for i, w in enumerate(vocab)}
    vocab_size = len(vocab)

    doc_ids = df["doc_id"].to_numpy()
    word_ids = np.array([word_to_id[w] for w in df["word"].to_list()])
    n_docs = int(doc_ids.max()) + 1

    rng = np.random.default_rng(seed)
    topic_ids, n_dk, n_kw, n_k = init_lda(
        doc_ids, word_ids, n_docs, n_topics, vocab_size, rng
    )
    for _ in range(n_iter):
        gibbs_step(
            doc_ids, word_ids, topic_ids, n_dk, n_kw, n_k, alpha, beta, vocab_size, rng
        )
    return n_dk, n_kw, vocab

Top Words Per Topic

  • After sampling, the top words for each topic are the words with the highest counts in the $n_{kw}$ matrix.
  • For this corpus, each topic's top words are drawn entirely from the ten-word vocabulary assigned to that generating topic.
def top_words(n_kw, vocab, n_words=5):
    """Return a list of n_topics lists, each containing the top n_words for that topic."""
    result = []
    for k in range(n_kw.shape[0]):
        indices = np.argsort(n_kw[k])[::-1][:n_words]
        result.append([vocab[i] for i in indices])
    return result

Document-Topic Proportions

  • The posterior estimate of document $d$'s topic mixture is obtained by smoothing the count row with $\alpha$ and normalising:

$$\hat{\theta}{dk} = \frac{n$$} + \alpha}{\sum_k (n_{dk} + \alpha)

def doc_topic_proportions(n_dk, alpha=ALPHA):
    """Return a (n_docs, n_topics) array of normalised document-topic proportions.

    Each row is smoothed with alpha before normalising:
      theta[d, k] = (n_dk[d, k] + alpha) / sum_k (n_dk[d, k] + alpha)
    Smoothing ensures that topics with zero observed tokens still receive a
    small non-zero probability rather than exactly zero.
    """
    smoothed = n_dk.astype(float) + alpha
    return smoothed / smoothed.sum(axis=1, keepdims=True)

Visualizing the Results

def plot_doc_topics(n_dk, true_topics, filename, alpha=ALPHA):
    """Save a heatmap of document-topic proportions sorted by true topic."""
    proportions = doc_topic_proportions(n_dk, alpha)
    n_docs, n_topics = proportions.shape
    rows = []
    for d in range(n_docs):
        label = f"doc {d:02d} (true {true_topics[d]})"
        for k in range(n_topics):
            rows.append(
                {
                    "document": label,
                    "topic": f"topic {k}",
                    "proportion": float(proportions[d, k]),
                    "sort_key": true_topics[d] * 100 + d,
                }
            )
    df = pl.DataFrame(rows)
    doc_order = (
        df.select(["document", "sort_key"])
        .unique()
        .sort("sort_key")["document"]
        .to_list()
    )
    chart = (
        alt.Chart(df)
        .mark_rect()
        .encode(
            x=alt.X("topic:N", title="Inferred topic"),
            y=alt.Y("document:N", title="Document", sort=doc_order),
            color=alt.Color(
                "proportion:Q",
                title="Proportion",
                scale=alt.Scale(scheme="blues"),
            ),
        )
        .properties(
            width=200,
            height=320,
            title="Document-topic proportions after Gibbs sampling",
        )
    )
    chart.save(filename)

Testing

Count matrix consistency

  • After init_lda, each row of $n_{kw}$ and each column of $n_{dk}$ must sum to the corresponding entry in $n_k$: every token is counted exactly once.

Proportions sum to one

  • After smoothing with $\alpha$ and normalising, each row of the proportion matrix must sum to 1.0.

Top-words order

  • Given a hand-crafted $n_{kw}$ with known highest-count words, top_words must return them in descending order.

Topic recovery

  • After running the sampler on the synthetic corpus, every document in the same true-topic group must share the same inferred dominant topic. The true-to-inferred mapping may be any permutation.
import numpy as np
from generate_topics import make_corpus, N_TOPICS, DOCS_PER_TOPIC
from topics import init_lda, run_lda, top_words, doc_topic_proportions


def test_count_matrices_consistent():
    # After init_lda, n_k must equal both the row sums of n_kw and the
    # column sums of n_dk: every token is counted exactly once in each matrix.
    df = make_corpus()
    vocab = sorted(df["word"].unique().to_list())
    vocab_size = len(vocab)
    word_to_id = {w: i for i, w in enumerate(vocab)}
    doc_ids = df["doc_id"].to_numpy()
    word_ids = np.array([word_to_id[w] for w in df["word"].to_list()])
    n_docs = int(doc_ids.max()) + 1
    rng = np.random.default_rng(0)
    _, n_dk, n_kw, n_k = init_lda(doc_ids, word_ids, n_docs, N_TOPICS, vocab_size, rng)
    np.testing.assert_array_equal(n_k, n_kw.sum(axis=1))
    np.testing.assert_array_equal(n_k, n_dk.sum(axis=0))


def test_doc_topic_proportions_sum_to_one():
    # Each document's topic proportions must sum to 1 after smoothing.
    n_dk = np.array([[10, 2, 0], [1, 8, 3]], dtype=int)
    props = doc_topic_proportions(n_dk, alpha=0.1)
    np.testing.assert_allclose(props.sum(axis=1), [1.0, 1.0])


def test_top_words_order():
    # Given a known n_kw, top_words must return the highest-count words first.
    n_kw = np.array([[100, 90, 1, 0], [0, 1, 80, 70]], dtype=int)
    vocab = ["w0", "w1", "w2", "w3"]
    result = top_words(n_kw, vocab, n_words=2)
    assert result[0] == ["w0", "w1"]
    assert result[1] == ["w2", "w3"]


def test_lda_recovers_topics():
    # After running the sampler, every document in the same true-topic group
    # must have the same inferred dominant topic.  (The mapping from true to
    # inferred topics may be permuted; the test only checks internal consistency.)
    df = make_corpus()
    n_dk, _, _ = run_lda(df, N_TOPICS)
    proportions = doc_topic_proportions(n_dk)
    assigned = np.argmax(proportions, axis=1)
    for k in range(N_TOPICS):
        group = assigned[k * DOCS_PER_TOPIC : (k + 1) * DOCS_PER_TOPIC]
        assert len(set(group.tolist())) == 1, (
            f"True-topic group {k} split across inferred topics: {group.tolist()}"
        )

Topic modeling key terms

Latent Dirichlet Allocation (LDA)
A generative probabilistic model in which each document is a mixture of topics and each topic is a distribution over words; both mixtures are drawn from Dirichlet priors with concentration parameters $\alpha$ and $\beta$
Collapsed Gibbs sampling
An MCMC algorithm that infers topic assignments by iteratively resampling each token's topic conditioned on all other assignments, with the document-topic and topic-word distributions integrated out (collapsed)
Bag of words
A document representation that records only which words appear and how often, discarding word order; sufficient for LDA because topics are defined by co-occurrence patterns rather than sequence
Dirichlet distribution
A distribution over probability vectors; used in LDA as the prior for both document-topic and topic-word distributions; small concentration parameters produce sparse (concentrated) distributions

Exercises

Effect of alpha and beta

Run the sampler with $\alpha \in {0.01, 0.1, 1.0}$ and $\beta \in {0.001, 0.01, 0.1}$ on the default corpus. For which combination of $\alpha$ and $\beta$ does the heatmap show the sharpest topic separation? Explain why large $\alpha$ blurs the document-topic proportions even when the corpus has clear topics.

Mixed documents

Modify make_corpus to generate five documents that draw 50% of their tokens from topic 0 and 50% from topic 1. Add those documents to the corpus and re-run the sampler. Do the mixed documents appear as a blend in the heatmap, or does the sampler assign them to a single topic?

Convergence check

Track the total number of topic reassignments per Gibbs step. Plot this count against iteration number. After how many iterations does the count stabilize? Compare the result for $\alpha = 0.01$ and $\alpha = 1.0$.

Perplexity

Implement a perplexity(held_out_df, n_dk, n_kw, alpha, beta) function that estimates the per-token perplexity of held-out documents: $\exp!\left(-\frac{1}{N}\sum_i \log P(w_i)\right)$ where $P(w_i) = \sum_k \hat{\theta}{d_ik} \hat{\phi}$. Lower perplexity indicates a better model. How does perplexity change as you increase the number of topics from 2 to 6?
