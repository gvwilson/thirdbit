Flood Frequency Analysis

The Problem

  • Infrastructure such as bridges, levees, and stormwater systems must be designed to withstand rare but extreme floods.
  • Engineers specify design events by return period: the average number of years between events that exceed a given magnitude.
  • A 100-year flood does not occur once per century on a schedule; it has a 1% probability of being exceeded in any given year.
  • Estimating these probabilities from a short historical record requires fitting a statistical distribution to the annual maximum flows.

The Gumbel Distribution

  • Annual maximum flows follow an extreme value distribution because they are the largest of many daily observations.
  • The Gumbel (Type I extreme value) distribution has cumulative distribution function:

$$F(x) = \exp\!\left(-\exp\!\left(-\frac{x - \mu}{\beta}\right)\right)$$

where $\mu$ is the location parameter (controls the centre) and $\beta > 0$ is the scale parameter (controls the spread).

  • Its mean and standard deviation are:

$$\mathrm{E}[X] = \mu + \gamma_E \beta \qquad \mathrm{SD}[X] = \frac{\pi \beta}{\sqrt{6}}$$

where $\gamma_E \approx 0.5772$ is the Euler-Mascheroni constant.

A river gauge records 50 years of annual maximum flows with sample mean 122 m^3/s and standard deviation 50 m^3/s. Using the Gumbel formulas above, what is $\hat{\beta}$?

39.0
Correct: $\hat{\beta} = 50 \times \sqrt{6} / \pi \approx 39.0$ m^3/s.
50.0
Wrong: 50 m^3/s is the sample standard deviation, not the scale parameter.
28.7
Wrong: dividing by $\pi$ without multiplying by $\sqrt{6}$ gives $50 / \pi \approx 15.9$, not this value.
122.0
Wrong: 122 m^3/s is the sample mean, which is used to estimate $\mu$, not $\beta$.

Generating Synthetic Data

  • We draw 50 years of annual maximum flows from a known Gumbel distribution so that we can verify our estimator.
  • The inverse-CDF method: if $U \sim \text{Uniform}(0, 1)$ then $X = \mu - \beta \ln(-\ln U)$ follows $\text{Gumbel}(\mu, \beta)$.
# True Gumbel parameters used to generate the synthetic flow record.
# The Gumbel distribution has mean = MU_TRUE + EULER * BETA_TRUE and
# standard deviation = pi * BETA_TRUE / sqrt(6), where EULER ~ 0.5772.
# With MU_TRUE = 100 and BETA_TRUE = 39 the mean is about 122 m^3/s
# and the standard deviation is about 50 m^3/s.
MU_TRUE = 100.0  # location parameter (m^3/s)
BETA_TRUE = 39.0  # scale parameter (m^3/s)

N_YEARS = 50  # years of annual maximum flow record

def generate(mu=MU_TRUE, beta=BETA_TRUE, n=N_YEARS, seed=SEED):
    """Return a DataFrame with synthetic annual maximum flows.

    Draws n values from a Gumbel distribution with the given location mu
    and scale beta.  Uses the inverse-CDF method: if U ~ Uniform(0, 1)
    then X = mu - beta * ln(-ln(U)) follows a Gumbel(mu, beta) distribution.
    """
    rng = np.random.default_rng(seed)
    u = rng.uniform(0.0, 1.0, n)
    flows = mu - beta * np.log(-np.log(u))
    return pl.DataFrame({"year": np.arange(1, n + 1), "annual_max_flow": flows})

Fitting by the Method of Moments

  • The method of moments sets the distribution's theoretical mean and variance equal to their sample counterparts and solves for the parameters.
  • From the Gumbel formulas, solving for $\hat{\beta}$ and $\hat{\mu}$:

$$\hat{\beta} = \frac{s\sqrt{6}}{\pi} \qquad \hat{\mu} = \bar{x} - \gamma_E \hat{\beta}$$

where $\bar{x}$ is the sample mean and $s$ is the sample standard deviation with Bessel's correction ($n-1$ denominator).

# Euler-Mascheroni constant: appears in the mean and quantile of the Gumbel
# distribution.  Value from Abramowitz & Stegun, §23.1.1.
EULER_MASCHERONI = 0.5772156649

# Return periods of interest (years).  A T-year flood has probability
# 1/T of being exceeded in any given year.
RETURN_PERIODS = [2, 5, 10, 25, 50, 100, 200]

def fit_gumbel(flows):
    """Estimate Gumbel location and scale by the method of moments.

    The Gumbel distribution satisfies:
      mean = mu + gamma * beta      (gamma = Euler-Mascheroni constant)
      std  = pi * beta / sqrt(6)

    Solving for the two unknowns given the sample mean and standard
    deviation yields the method-of-moments estimates.
    """
    mean = float(np.mean(flows))
    std = float(np.std(flows, ddof=1))
    beta_hat = std * np.sqrt(6.0) / np.pi
    mu_hat = mean - EULER_MASCHERONI * beta_hat
    return mu_hat, beta_hat

Put these steps in the correct order for method-of-moments estimation of Gumbel parameters.

  1. Compute the sample mean and sample standard deviation of the annual maximum flows
  2. Estimate the scale parameter: beta_hat = std * sqrt(6) / pi
  3. Estimate the location parameter: mu_hat = mean - EULER_MASCHERONI * beta_hat
  4. Use mu_hat and beta_hat to compute return levels or plot the fitted distribution

Return Periods

  • The $T$-year return level $x_T$ is the flow magnitude exceeded with probability $1/T$ in any given year.
  • Setting $F(x_T) = 1 - 1/T$ and solving the Gumbel CDF for $x_T$:

$$x_T = \hat{\mu} - \hat{\beta} \ln\!\left(-\ln\!\left(1 - \frac{1}{T}\right)\right)$$

  • Equivalently, $x_T = \hat{\mu} + \hat{\beta}\, y_T$, where $y_T = -\ln(-\ln(1-1/T))$ is the reduced variate at return period $T$.
def return_level(mu, beta, T):
    """Return the flood magnitude exceeded on average once every T years.

    The T-year return level is the quantile at non-exceedance probability
    p = 1 - 1/T.  For the Gumbel distribution this is:
      x_T = mu - beta * ln(-ln(1 - 1/T))
    Equivalently, x_T = mu + beta * y_T, where y_T = -ln(-ln(p)) is the
    reduced variate at return period T.
    """
    p = 1.0 - 1.0 / T
    return mu - beta * np.log(-np.log(p))

Using $\hat{\mu} = 100.0$ m^3/s and $\hat{\beta} = 39.0$ m^3/s, compute the 100-year return level $x_{100} = \hat{\mu} - \hat{\beta} \ln(-\ln(1 - 1/100))$. Give your answer in m^3/s, rounded to one decimal place.

A community's flood barrier is designed for the 100-year return level. Over the next 100 years, what is the probability that the barrier is overtopped at least once?

Exactly 1%
Wrong: 1% is the probability of exceedance in a single year, not over 100 years.
Approximately 63%
Correct: P(at least one exceedance in 100 years) = 1 - (1 - 1/100)^100 ≈ 1 - e^{-1} ≈ 63%.
Exactly 100%
Wrong: the return period is an average; exceedance is a random event, not guaranteed over any finite horizon.
Exactly 50%
Wrong: the return period T such that P(at least one exceedance in T years) = 0.5 is T ≈ 69 years, not 100.

Gumbel Probability Paper

  • Plotting data on Gumbel probability paper gives a visual check on whether the Gumbel distribution fits.
  • The Weibull plotting position $p_i = i/(n+1)$ assigns a non-exceedance probability to each ranked observation without assuming a specific distribution.
  • The reduced variate $y_i = -\ln(-\ln(p_i))$ linearises the Gumbel CDF: if the data are Gumbel-distributed, the points $(y_i, x_{(i)})$ scatter around a straight line.
def plotting_positions(flows):
    """Return (reduced_variate, sorted_flows) for a Gumbel probability plot.

    Ranks observations from smallest to largest and assigns Weibull
    plotting positions p_i = i / (n + 1), which are approximately
    unbiased for the Gumbel distribution.  The reduced variate
    y_i = -ln(-ln(p_i)) linearises the Gumbel CDF: if the data follow
    a Gumbel distribution, plotting (y_i, x_(i)) gives a straight line.
    """
    n = len(flows)
    sorted_flows = np.sort(flows)
    ranks = np.arange(1, n + 1)
    p = ranks / (n + 1.0)
    y = -np.log(-np.log(p))
    return y, sorted_flows

def plot(flows, mu, beta):
    """Return an Altair Gumbel probability paper chart.

    Points are the empirical data; the line is the fitted Gumbel distribution.
    Both are plotted against the reduced variate y = -ln(-ln(p)).
    If the data are Gumbel-distributed, the points should scatter around
    a straight line.
    """
    y_obs, q_obs = plotting_positions(flows)
    y_fit = np.linspace(y_obs.min() - 0.5, y_obs.max() + 0.5, 200)
    x_fit = mu + beta * y_fit  # Gumbel quantile function: x = mu + beta * y

    df_pts = pl.DataFrame({"y": y_obs, "flow": q_obs})
    df_line = pl.DataFrame({"y": y_fit, "flow": x_fit})

    points = (
        alt.Chart(df_pts)
        .mark_point()
        .encode(
            x=alt.X("y:Q", title="Reduced variate  y = -ln(-ln p)"),
            y=alt.Y("flow:Q", title="Annual maximum flow (m^3/s)"),
        )
    )
    line = (
        alt.Chart(df_line)
        .mark_line(color="firebrick")
        .encode(
            x=alt.X("y:Q"),
            y=alt.Y("flow:Q"),
        )
    )
    return (points + line).properties(
        width=400, height=300, title="Gumbel probability paper"
    )

Testing

Parameter recovery on a large sample

  • With $n = 10\,000$ the central-limit theorem says the standard error of $\hat{\beta}$ is roughly $\beta/\sqrt{n}$, well under 1%.
  • A relative tolerance of 5% gives a safety factor of $\sim 5$ over the expected sampling error.

Monotone return levels

  • The Gumbel quantile $x_T = \mu + \beta y_T$ is strictly increasing in $T$ because $\beta > 0$ and $y_T$ is strictly increasing.
  • Any violation would indicate an error in the formula or the sign conventions.

Algebraic consistency

  • $F(x_T) = 1 - 1/T$ is the definition of the return level; it must hold to numerical precision ($<10^{-10}$).

Valid plotting positions

  • $p_i = i/(n+1) \in (0,1)$ guarantees finite reduced variates; sorted flows must be non-decreasing.
import numpy as np
from generate_flood import BETA_TRUE, MU_TRUE, generate
from flood import (
    fit_gumbel,
    plotting_positions,
    return_level,
)


def test_fit_gumbel_large_sample():
    # With N = 10 000 the central-limit theorem says the standard error of
    # the sample mean is roughly sigma / sqrt(N) ~ 50 / 100 = 0.5 m^3/s.
    # A relative tolerance of 5% (about 5 m^3/s for mu, 2 m^3/s for beta)
    # gives a safety factor of ~10 over this standard error.
    rng = np.random.default_rng(7493418)
    u = rng.uniform(0.0, 1.0, 10_000)
    flows = MU_TRUE - BETA_TRUE * np.log(-np.log(u))
    mu_hat, beta_hat = fit_gumbel(flows)
    assert abs(mu_hat - MU_TRUE) / MU_TRUE < 0.05
    assert abs(beta_hat - BETA_TRUE) / BETA_TRUE < 0.05


def test_return_levels_increase_with_period():
    # A T-year flood is rarer and more extreme than a T/2-year flood, so
    # return levels must be strictly increasing in T.  The Gumbel quantile
    # x_T = mu + beta * (-ln(-ln(1 - 1/T))) is strictly increasing because
    # beta > 0 and -ln(-ln(1 - 1/T)) is strictly increasing in T.
    flows = generate()["annual_max_flow"].to_numpy()
    mu, beta = fit_gumbel(flows)
    levels = [return_level(mu, beta, T) for T in [2, 10, 50, 100, 500]]
    assert all(levels[i] < levels[i + 1] for i in range(len(levels) - 1))


def test_return_level_probability():
    # By construction, x_T satisfies F(x_T) = 1 - 1/T under the Gumbel CDF
    # F(x) = exp(-exp(-(x - mu) / beta)).  This is an algebraic identity, so
    # the residual should be below floating-point machine epsilon (~1e-15).
    mu, beta = 100.0, 39.0
    T = 100
    x_T = return_level(mu, beta, T)
    p_fitted = np.exp(-np.exp(-(x_T - mu) / beta))
    assert abs(p_fitted - (1.0 - 1.0 / T)) < 1e-10


def test_plotting_positions_valid():
    # Weibull positions i/(n+1) lie in (0, 1), so the reduced variate
    # y = -ln(-ln(p)) is always finite.  Sorted flows must be non-decreasing.
    flows = generate()["annual_max_flow"].to_numpy()
    y, q = plotting_positions(flows)
    assert np.isfinite(y).all(), "reduced variates contain infinite values"
    assert (np.diff(q) >= 0).all(), "flows are not sorted ascending"

Exercises

Confidence intervals by bootstrap

Estimate 95% confidence intervals for the 100-year return level using the bootstrap: draw 1000 samples of size 50 with replacement from the original data, fit the Gumbel distribution to each, and compute the 100-year return level. Report the 2.5th and 97.5th percentiles of the 1000 estimates. How wide is the interval relative to the point estimate?

Gringorten plotting positions

Replace the Weibull plotting position $p_i = i/(n+1)$ with the Gringorten position $p_i = (i - 0.44) / (n + 0.12)$, which is designed to be approximately unbiased for the Gumbel distribution. How much do the plotted points shift on the probability paper? Does the fitted line change?

Comparing to the Generalised Extreme Value distribution

The Gumbel distribution is a special case (shape $\xi = 0$) of the three-parameter Generalised Extreme Value (GEV) distribution. Use scipy.stats.genextreme.fit to fit the GEV to the same 50-year record. Does the estimated shape parameter differ significantly from zero? Compare the 100-year return level from the GEV with the Gumbel estimate.

Annual maxima from daily data

Download or generate a synthetic series of daily flows for 50 years (e.g., a log-normal random walk with autocorrelation). Extract the annual maximum from each year. Fit the Gumbel distribution to the maxima and plot the result on probability paper. How does the quality of the fit compare to the case where the data were drawn directly from a Gumbel distribution?
