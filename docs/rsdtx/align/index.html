Pairwise Sequence Alignment

The Problem

  • Two DNA sequences often share regions of similarity because they evolved from a common ancestor or perform a similar function.
  • Identifying those shared regions by eye is impractical for sequences longer than a few dozen characters.
  • Sequence alignment finds the best correspondence between two sequences automatically, allowing mismatches and gaps.
  • A gap (represented as -) means a character is present in one sequence but absent in the other, which corresponds to an insertion or deletion during evolution.
  • Local alignment finds the highest-scoring matching subsequence anywhere in the two sequences, rather than aligning them end to end.

Scoring an Alignment

  • Every aligned pair of positions contributes to a total score:
  • matching characters contribute a positive reward,
  • mismatched characters contribute a small penalty (keeping mismatches is sometimes better than introducing gaps),
  • each gap character contributes a larger penalty.
MATCH_SCORE = 2  # reward for aligning identical characters
MISMATCH_PENALTY = -1  # penalty for aligning different characters
GAP_PENALTY = -2  # penalty per gap character inserted in either sequence

# Example sequences for the lesson.
SEQ_A = "ATCGT"
SEQ_B = "ATGT"
  • MATCH_SCORE > 0 and GAP_PENALTY < MISMATCH_PENALTY < 0 because a gap represents a larger evolutionary event than a point substitution.

The Smith-Waterman Algorithm

  • Smith-Waterman (1981) solves local alignment exactly using dynamic programming.
  • Build a matrix $H$ of size $(m+1)\times(n+1)$ where $m$ and $n$ are the sequence lengths.
  • The extra row and column hold zeros that represent alignments of empty prefixes.
  • Fill $H$ left-to-right, top-to-bottom using the recurrence:

$$H(i,j) = \max\!\begin{cases} 0 \\ H(i-1,\,j-1) + s(a_i,\,b_j) \\ H(i-1,\,j) + g \\ H(i,\,j-1) + g \end{cases}$$

where $s(a_i, b_j)$ is MATCH_SCORE or MISMATCH_PENALTY and $g$ is GAP_PENALTY. - The zero case is the key difference from global alignment (Needleman-Wunsch): it means a local alignment can start anywhere, ignoring unfavorable prefixes.

def _nt_score(a, b):
    """Return match score or mismatch penalty for two nucleotide characters."""
    return MATCH_SCORE if a == b else MISMATCH_PENALTY

def fill_matrix(seq_a, seq_b):
    """Return the Smith-Waterman scoring matrix for two sequences.

    The matrix has shape (len(seq_a)+1, len(seq_b)+1).  Entry H[i, j] holds
    the score of the best local alignment ending at seq_a[i-1] and seq_b[j-1].
    All entries are non-negative: the algorithm ignores alignments that would
    produce a negative running total.
    """
    m, n = len(seq_a), len(seq_b)
    H = np.zeros((m + 1, n + 1), dtype=int)
    for i in range(1, m + 1):
        for j in range(1, n + 1):
            diag = H[i - 1, j - 1] + _nt_score(seq_a[i - 1], seq_b[j - 1])
            delete = H[i - 1, j] + GAP_PENALTY  # gap in seq_b
            insert = H[i, j - 1] + GAP_PENALTY  # gap in seq_a
            H[i, j] = max(0, diag, delete, insert)
    return H

Smith-Waterman uses max(0, …) in its recurrence. What is the effect of this zero floor?

It prevents integer overflow when sequences are long
Wrong: the scores stay small regardless of sequence length; overflow is not a concern here.
It allows the alignment to restart anywhere, enabling local (subsequence) alignment
Correct: resetting to zero means a new local alignment can begin at any cell, so only the best-scoring subsequence is reported.
It speeds up the fill step by skipping zero cells
Wrong: every cell is still filled; the zero floor changes the value stored, not whether computation occurs.
It forces gaps to be penalised more heavily than mismatches
Wrong: the relative penalty sizes are set by GAP_PENALTY and MISMATCH_PENALTY, not by the zero floor.

Traceback

  • The cell with the highest value in $H$ is the end of the best local alignment.
  • Follow the path that produced each cell's score back through $H$ until reaching a zero.
  • Prefer the diagonal move (match/mismatch) over delete or insert when scores are tied, to produce a unique result.
  • Append a gap character when the path goes up (gap in $b$) or left (gap in $a$).
  • Reverse the collected characters at the end, since the traceback runs backwards.
def traceback(H, seq_a, seq_b):
    """Trace back from the highest-scoring cell to recover the alignment.

    Returns (aligned_a, aligned_b, score) where '-' marks a gap.
    When the diagonal, delete, and insert moves give equal scores, the
    diagonal (match/mismatch) move is preferred to ensure a unique result.
    """
    i, j = np.unravel_index(np.argmax(H), H.shape)
    score = int(H[i, j])
    aligned_a, aligned_b = [], []
    while H[i, j] > 0:
        if (
            i > 0
            and j > 0
            and H[i, j] == H[i - 1, j - 1] + _nt_score(seq_a[i - 1], seq_b[j - 1])
        ):
            aligned_a.append(seq_a[i - 1])
            aligned_b.append(seq_b[j - 1])
            i -= 1
            j -= 1
        elif i > 0 and H[i, j] == H[i - 1, j] + GAP_PENALTY:
            aligned_a.append(seq_a[i - 1])
            aligned_b.append("-")
            i -= 1
        else:
            aligned_a.append("-")
            aligned_b.append(seq_b[j - 1])
            j -= 1
    return "".join(reversed(aligned_a)), "".join(reversed(aligned_b)), score

def align(seq_a, seq_b):
    """Return the best local alignment of seq_a and seq_b as (aligned_a, aligned_b, score)."""
    H = fill_matrix(seq_a, seq_b)
    return traceback(H, seq_a, seq_b)

Put these Smith-Waterman steps in the correct order.

  1. Initialise an (m+1) X (n+1) matrix of zeros
  2. Fill the matrix left-to-right and top-to-bottom using the recurrence
  3. Find the cell with the maximum score
  4. Follow the traceback path until reaching a cell with score 0
  5. Reverse the collected characters to produce the aligned sequences

Label each traceback move with the alignment operation it represents.

Traceback Move Alignment Operation
Diagonal move (↖) Match or mismatch between one character from each sequence
Upward move (↑) Gap inserted in sequence B (character in A has no counterpart in B)
Leftward move (←) Gap inserted in sequence A (character in B has no counterpart in A)
Stop (score = 0) End of the local alignment — do not continue the traceback

Displaying the Alignment

def format_alignment(aligned_a, aligned_b):
    """Return a three-line string representation of an alignment.

    The middle row marks matches with '|' and mismatches or gaps with ' '.
    """
    middle = "".join("|" if a == b else " " for a, b in zip(aligned_a, aligned_b))
    return f"a: {aligned_a}\n   {middle}\nb: {aligned_b}"

Running the algorithm on SEQ_A = "ATCGT" and SEQ_B = "ATGT" produces:

a: ATCGT
   || ||
b: AT-GT

The gap in seq_b at position 3 accounts for the C in seq_a that has no counterpart. The score is $4 \times 2 + (-2) = 6$: four matches at MATCH_SCORE = 2 and one gap at GAP_PENALTY = −2.

Visualizing the Scoring Matrix

  • Plotting $H$ as a heatmap shows where high-scoring regions form.
  • Bright cells cluster along diagonals where the sequences share runs of matching characters.
def plot_matrix(H, seq_a, seq_b):
    """Return an Altair heatmap of the scoring matrix."""
    rows = [
        {
            "i": i,
            "j": j,
            "score": int(H[i, j]),
            "row_label": seq_a[i - 1] if i > 0 else "",
            "col_label": seq_b[j - 1] if j > 0 else "",
        }
        for i in range(H.shape[0])
        for j in range(H.shape[1])
    ]
    df = pl.DataFrame(rows)
    return (
        alt.Chart(df)
        .mark_rect()
        .encode(
            x=alt.X(
                "j:O",
                title=f"seq_b: {seq_b}",
                axis=alt.Axis(labelExpr="datum.value == 0 ? '' : datum.value"),
            ),
            y=alt.Y(
                "i:O",
                title=f"seq_a: {seq_a}",
                sort="descending",
                axis=alt.Axis(labelExpr="datum.value == 0 ? '' : datum.value"),
            ),
            color=alt.Color("score:Q", scale=alt.Scale(scheme="blues"), title="Score"),
            tooltip=["i", "j", "score"],
        )
        .properties(width=300, height=300, title="Smith-Waterman scoring matrix")
    )

Testing

Unlike the diffusion and Lotka-Volterra lessons, all scores here are integers computed by exact arithmetic — no floating-point rounding occurs and no tolerance is needed in any test.

Matrix shape

  • The matrix must have one extra row and column for the zero border.
  • An off-by-one error in the loop bounds is the most common bug in DP implementations.

No negative scores

  • The max(0, ...) in the recurrence guarantees every cell is non-negative.
  • A negative value means the zero floor was accidentally omitted.

Identical sequences

  • Aligning a sequence with itself must match at every position with no gaps or mismatches.
  • The score is exactly $\text{len} \times \text{MATCH_SCORE}$.
  • This test also checks that the aligned strings are returned in the correct order (not reversed).

Mismatch in alignment

  • "AAAC" vs "AGAC": the second character differs (A vs G), but flanking matches make including the mismatch better than starting a new alignment.
  • Score = $3 \times 2 + (-1) = 5$.
  • This test verifies that the algorithm correctly prefers a gapless alignment with one mismatch over breaking the alignment at the mismatch.

Gap in alignment

  • "ATCGT" vs "ATGT": seq_b has no C, so the aligner inserts a gap.
  • Score = $4 \times 2 + (-2) = 6$.
  • This test is the key check on the gap-handling branch of the traceback.

No shared characters

  • When every pair of characters mismatches, all cells in $H$ are zero (the floor prevents any negative score from propagating).
  • The traceback finds a maximum of zero and returns immediately with empty strings.
from align import (
    GAP_PENALTY,
    MATCH_SCORE,
    MISMATCH_PENALTY,
    SEQ_A,
    SEQ_B,
    align,
    fill_matrix,
)


def test_matrix_shape():
    # The scoring matrix must have one extra row and column for the
    # zero-initialized border that represents alignments of empty prefixes.
    H = fill_matrix(SEQ_A, SEQ_B)
    assert H.shape == (len(SEQ_A) + 1, len(SEQ_B) + 1)


def test_no_negative_scores():
    # Smith-Waterman replaces any negative running total with zero, so every
    # cell must be non-negative.  A negative value would indicate a bug in the
    # max(0, ...) application.
    H = fill_matrix(SEQ_A, SEQ_B)
    assert (H >= 0).all()


def test_identical_sequences():
    # A sequence aligned with itself must match at every position.  The score
    # is exactly len(seq) * MATCH_SCORE with no gaps or mismatches.
    # Integer arithmetic means no tolerance is needed.
    seq = "ACGT"
    aligned_a, aligned_b, score = align(seq, seq)
    assert aligned_a == seq
    assert aligned_b == seq
    assert score == len(seq) * MATCH_SCORE


def test_mismatch_in_alignment():
    # "AAAC" vs "AGAC": the second character is a mismatch (A vs G) but the
    # surrounding matches make it worthwhile to include rather than starting
    # a new alignment.  Score = 3*MATCH_SCORE + MISMATCH_PENALTY = 5.
    aligned_a, aligned_b, score = align("AAAC", "AGAC")
    assert aligned_a == "AAAC"
    assert aligned_b == "AGAC"
    assert score == 3 * MATCH_SCORE + MISMATCH_PENALTY


def test_gap_in_alignment():
    # "ATCGT" vs "ATGT": the C in seq_a has no counterpart in seq_b.
    # The best alignment inserts a gap in seq_b rather than breaking the
    # flanking matches.  Score = 4*MATCH_SCORE + GAP_PENALTY = 6.
    aligned_a, aligned_b, score = align("ATCGT", "ATGT")
    assert aligned_a == "ATCGT"
    assert aligned_b == "AT-GT"
    assert score == 4 * MATCH_SCORE + GAP_PENALTY


def test_no_shared_characters():
    # When every pair of characters is a mismatch, every cell in H is 0
    # (the zero floor prevents any negative score from propagating).
    # The traceback starts at the maximum, which is 0, and returns immediately,
    # producing empty aligned strings.
    aligned_a, aligned_b, score = align("AAAA", "CCCC")
    assert aligned_a == ""
    assert aligned_b == ""
    assert score == 0

Exercises

Effect of gap penalty

Increase GAP_PENALTY from −2 to −1 (equal to MISMATCH_PENALTY) and re-run align("ATCGT", "ATGT"). Does the alignment still contain a gap, or does it change? Explain the result in terms of the recurrence relation.

Global alignment (Needleman-Wunsch)

The Needleman-Wunsch algorithm for global alignment differs from Smith-Waterman in exactly one way: cells in the first row and column are initialized to $0, g, 2g, 3g, \ldots$ instead of all zeros, and the max(0, ...) floor is removed. Implement fill_global(seq_a, seq_b) and traceback_global(H, seq_a, seq_b). Verify that aligning "ACGT" with "ACT" globally produces a different result than aligning them locally.

Scoring matrix for amino acids

Biological sequence aligners use substitution matrices such as BLOSUM62 that assign different scores to each pair of amino acid characters, reflecting how often each substitution occurs in evolution. Replace _nt_score with a function that looks up scores in a small hand-written $4 \times 4$ substitution matrix for the four DNA bases A, C, G, T. Assign a higher score to transitions (A↔G, C↔T) than transversions (A↔C, A↔T, etc.), since transitions are more common mutations. Verify that aligning "ATCG" with "AGCG" now gives a higher score than with the flat MATCH_SCORE / MISMATCH_PENALTY scheme.

Affine gap penalty

The linear gap penalty $g$ treats opening a gap and extending an existing gap identically. A more realistic affine gap penalty charges a higher cost $g_o$ to open a gap and a lower cost $g_e$ to extend it: a gap of length $k$ costs $g_o + (k-1)g_e$. Look up the three-matrix formulation of Smith-Waterman with affine gaps (using matrices $H$, $E$, $F$) and implement it. Verify that with $g_o = -3$ and $g_e = -1$ the alignment of "ATCCCGT" with "ATGT" produces a single gap of length 3 rather than three individual gaps.
