Pollutant Spread in a River Network

The Problem

  • A chemical spill at a river headwater contaminates the water supply of communities downstream.
  • River networks branch and merge: pollutant introduced at one node is carried to all nodes reachable from it.
  • We want to track the concentration at each node over time to predict when downstream communities are affected.
  • This is a flow network problem: a directed graph where edges carry material from node to node.

Representing the River Network

  • The network is a directed tree: water flows from headwaters to an outlet, with no cycles.
  • Nodes represent measurement stations or junctions; edges represent river reaches.
  • Edge weights record relative discharge (mean flow rate), which governs both dilution and visualization.
# River network: 5 nodes in a directed tree (upstream to downstream).
# Node indices correspond to NODE_NAMES.
NODE_NAMES = ["upstream_a", "upstream_b", "junction", "reach", "outlet"]

# Node 4 (outlet) is the sink: pollutant that arrives there is not transported further.
SINK = 4

# Initial spill: unit concentration at upstream_a only.
SPILL_NODE = 0

# Fraction of concentration at each non-sink node transported downstream per step.
# At 0.4, roughly 40% of mass moves forward each time step.
TRANSPORT_RATE = 0.4

N_STEPS = 20
SNAPSHOT_INTERVAL = 5

def build_network():
    """Return a directed graph representing the river network.

    Edge weights represent relative mean discharge (arbitrary units) and
    are used for visualization only; transport uses a uniform rate.
    """
    G = nx.DiGraph()
    G.add_nodes_from(range(len(NODE_NAMES)))
    # upstream_a and upstream_b both drain into the junction.
    G.add_edge(0, 2, weight=3.0)
    G.add_edge(1, 2, weight=5.0)
    # junction drains to reach, reach drains to outlet.
    G.add_edge(2, 3, weight=8.0)
    G.add_edge(3, 4, weight=8.0)
    return G
  • upstream_a (node 0) is the spill site; upstream_b (node 1) is an uncontaminated headwater.
  • Both drain into junction (node 2), which drains to reach (node 3), which drains to outlet (node 4).
  • Node 4 is the sink: pollutant that arrives there accumulates and is not transported further.

Why is a river network represented as a directed graph rather than an undirected graph?

Water can flow in either direction between any two connected nodes.
Wrong: rivers flow in one direction, from higher to lower elevation; edges must be directed to encode this.
The direction of each edge encodes which way water (and pollutant) flows.
Correct: directed edges from upstream to downstream nodes capture the unidirectional nature of river flow.
Directed graphs are always faster to compute with than undirected graphs.
Wrong: the choice of directed vs. undirected is based on the physical system, not computational speed.
Edge weights cannot be stored on undirected graphs.
Wrong: undirected graphs support weighted edges just as directed graphs do.

The Transport Model

  • At each time step, a fraction TRANSPORT_RATE of the concentration at each non-sink node moves downstream.
  • The remaining fraction $(1 - \text{TRANSPORT_RATE})$ stays at the node (representing mixing and retention in the water column).
  • For node $j$ with downstream successor $i$, the update rule is:

$$c_i^{n+1} = c_i^n + \alpha \, c_j^n \qquad c_j^{n+1} = (1 - \alpha) \, c_j^n$$

where $\alpha = \text{TRANSPORT_RATE}$.

  • Combined across all nodes, this is a single matrix-vector multiplication $\mathbf{c}^{n+1} = T\,\mathbf{c}^n$.
  • The transfer matrix $T$ encodes the entire network topology and transport rate in one object.

Building the Transfer Matrix

def make_transfer(G):
    """Return the N x N transfer matrix for one time step.

    T[i, j] is the fraction of the concentration at node j that arrives
    at node i in one step.  Each non-sink node retains (1 - TRANSPORT_RATE)
    of its concentration and sends TRANSPORT_RATE to its downstream
    successor(s), divided equally when there are several.  The sink
    retains everything it receives (T[sink, sink] = 1).

    Because every column of T sums to 1, mass is exactly conserved:
    the total concentration summed over all nodes never changes.
    """
    n = len(NODE_NAMES)
    T = np.eye(n)  # default: each node retains everything
    for j in range(n):
        if j == SINK:
            continue
        succs = list(G.successors(j))
        if not succs:
            continue
        T[j, j] = 1.0 - TRANSPORT_RATE
        for i in succs:
            T[i, j] = TRANSPORT_RATE / len(succs)
    return T
  • The matrix starts as the identity: every node retains everything.
  • Non-sink nodes are then modified to lose TRANSPORT_RATE to their successor(s).
  • Each column of $T$ sums to 1: the total concentration (mass) is exactly conserved at every step.

Put these steps in the correct order to build one column of the transfer matrix for a non-sink node j.

  1. Identify the list of downstream successors of j in the graph G
  2. Set T[j, j] = 1 - TRANSPORT_RATE (fraction retained at j)
  3. For each successor i, set T[i, j] = TRANSPORT_RATE / number_of_successors
  4. Verify that the column sums to 1 (mass conservation check)

In our 5-node tree every non-sink node has exactly one downstream successor. What does TRANSPORT_RATE / len(succs) equal in this case?

0.0
Wrong: that would mean no pollutant is ever transported.
TRANSPORT_RATE / 2
Wrong: len(succs) is 1, not 2, for nodes with a single successor.
TRANSPORT_RATE
Correct: dividing by 1 leaves TRANSPORT_RATE unchanged; all transported mass goes to the single successor.
1.0
Wrong: 1.0 would mean all concentration moves downstream and none is retained.

Running the Simulation

def simulate(G):
    """Simulate pollutant transport and return concentration snapshots.

    Advances c = T @ c one step at a time and records the state at
    step 0 and every SNAPSHOT_INTERVAL steps thereafter.
    """
    n = len(NODE_NAMES)
    c = np.zeros(n)
    c[SPILL_NODE] = 1.0
    T = make_transfer(G)

    records = _snapshot(c, 0)
    for step in range(1, N_STEPS + 1):
        c = T @ c
        if step % SNAPSHOT_INTERVAL == 0:
            records += _snapshot(c, step)
    return pl.DataFrame(records)


def _snapshot(c, step):
    return [
        {"node": NODE_NAMES[i], "concentration": float(c[i]), "step": step}
        for i in range(len(NODE_NAMES))
    ]
  • The loop applies the transfer matrix repeatedly, saving a snapshot every SNAPSHOT_INTERVAL steps.
  • Each snapshot records the concentration at all nodes at a given time step.

Visualizing the Results

def plot(df):
    """Return an Altair line chart of concentration at each node over time."""
    return (
        alt.Chart(df)
        .mark_line(point=True)
        .encode(
            x=alt.X("step:O", title="Time step"),
            y=alt.Y("concentration:Q", title="Concentration",
                    scale=alt.Scale(domain=[0.0, 1.0])),
            color=alt.Color("node:N", title="Node", sort=NODE_NAMES),
        )
        .properties(width=350, height=250)
    )

Testing

Mass conservation

  • The transfer matrix is column-stochastic (columns sum to 1), so $\sum_i c_i^n = \sum_i c_i^0 = 1$ at every step.
  • This holds exactly in floating-point arithmetic for this matrix structure; no tolerance is needed beyond rounding ($10^{-12}$).

Non-negativity

  • All entries of $T$ are non-negative and each update is a convex combination of non-negative values.
  • Concentration cannot go negative; any negative value would indicate a code error.

Monotone decay at the spill node

  • upstream_a has no upstream predecessors, so its column of $T$ has only the diagonal entry $(1 - \alpha)$.
  • Starting from $c_0 = 1$, concentration there decays as $(1 - \alpha)^n$, strictly decreasing.

Uncontaminated node stays zero

  • upstream_b also has no predecessors and starts at 0.
  • Its concentration is $(1 - \alpha)^n \times 0 = 0$ at every step — a structural check on the network topology.

Monotone accumulation at the outlet

  • The outlet (sink) only receives mass and never releases it, so its concentration is non-decreasing.
import numpy as np
from pollute import (
    NODE_NAMES,
    N_STEPS,
    SINK,
    SPILL_NODE,
    TRANSPORT_RATE,
    build_network,
    make_transfer,
    simulate,
)


def test_mass_conservation():
    # The transfer matrix is column-stochastic: each column sums to exactly 1.
    # Starting from total mass = 1, the sum is conserved at every step in
    # exact floating-point arithmetic (no rounding accumulation from addition).
    G = build_network()
    T = make_transfer(G)
    c = np.zeros(len(NODE_NAMES))
    c[SPILL_NODE] = 1.0
    for _ in range(N_STEPS):
        c = T @ c
    assert abs(c.sum() - 1.0) < 1e-12


def test_all_nonnegative():
    # Each update c = T @ c is a convex combination of non-negative values
    # (all entries of T are non-negative), so no concentration can go negative.
    G = build_network()
    T = make_transfer(G)
    c = np.zeros(len(NODE_NAMES))
    c[SPILL_NODE] = 1.0
    for step in range(N_STEPS):
        c = T @ c
        assert (c >= 0).all(), f"negative concentration at step {step + 1}"


def test_spill_node_decreases():
    # upstream_a (node 0) has no upstream predecessors, so its concentration
    # decays by factor (1 - TRANSPORT_RATE) each step.  It must be strictly
    # decreasing from its initial value of 1.
    G = build_network()
    T = make_transfer(G)
    c = np.zeros(len(NODE_NAMES))
    c[SPILL_NODE] = 1.0
    prev = float(c[SPILL_NODE])
    for _ in range(N_STEPS):
        c = T @ c
        assert c[SPILL_NODE] < prev
        prev = float(c[SPILL_NODE])


def test_uncontaminated_node_stays_zero():
    # upstream_b (node 1) starts at zero and has no upstream predecessors.
    # Column 1 of T has T[1,1] = 1 - TRANSPORT_RATE and T[j,1] = 0 for j != 1.
    # Starting from c[1] = 0, it stays exactly zero at every step.
    G = build_network()
    T = make_transfer(G)
    c = np.zeros(len(NODE_NAMES))
    c[SPILL_NODE] = 1.0
    for _ in range(N_STEPS):
        c = T @ c
        assert c[1] == 0.0  # upstream_b is never contaminated


def test_outlet_nondecreasing():
    # Pollutant accumulates at the sink: c[SINK] increases whenever the reach
    # (node 3) has positive concentration, and never decreases.
    G = build_network()
    T = make_transfer(G)
    c = np.zeros(len(NODE_NAMES))
    c[SPILL_NODE] = 1.0
    prev = float(c[SINK])
    for _ in range(N_STEPS):
        c = T @ c
        assert c[SINK] >= prev
        prev = float(c[SINK])

Exercises

Adding a tributary

Add a sixth node tributary (node 5) that joins at reach (node 3) with edge weight 2.0. Update build_network and rerun the simulation. Does test_mass_conservation still pass? Does the pollutant reach the outlet faster or slower?

Variable transport rate

Replace the uniform TRANSPORT_RATE with per-edge rates stored in a dictionary keyed by (src, dst). Assign higher rates to high-flow edges (e.g., rate proportional to edge weight / max edge weight). Modify make_transfer to use per-edge rates. How does the arrival time at the outlet change compared to the uniform-rate model?

Continuous spill

Change the initial condition so that upstream_a receives a constant input of 0.1 concentration units per step (a continuous source rather than an instantaneous spill). Modify simulate to add this source term after each matrix update. Is mass still conserved? At what step does the outlet concentration stabilise?

Two-spill superposition

Run two separate simulations: one with the spill at upstream_a and one with a spill at upstream_b. Add the two concentration arrays at each step. Compare the result to a single simulation that starts with concentration 1.0 at both headwater nodes simultaneously. Are the results identical? Explain why (or why not) in terms of the linearity of the transfer matrix update.
